# 模仿者游戏 - 详细玩法流程报告

## 一、总览

模仿者游戏是移植自第五人格「模仿者游戏」模式的 Paper 1.21.5 插件。这是一款 10 人社交推理游戏，类似 Among Us，三方阵营各怀目标。

**标准模式（10人）：** 6侦探团 + 2模仿者 + 2神秘客

**核心循环：**

```
大厅准备 → 倒计时 → 身份揭示 → 自由行动 ⇄ 会议阶段 → 胜利判定 → 结束
```

---

## 二、完整游戏流程（分阶段详解）

### 阶段 0：大厅 (LOBBY)

**进入条件：** 插件加载后，游戏自动处于大厅状态。

**玩家操作：**
- 输入 `/ig join` 加入大厅
- 输入 `/ig leave` 离开大厅
- 输入 `/ig list` 查看当前大厅玩家列表

**关键细节：**
| 细节 | 说明 |
|------|------|
| 加入方式 | 仅命令加入，无自动匹配。必须玩家手动 `/ig join` |
| 人数限制 | 最低 10 人（`config.yml` 中 `min-players` 可配置） |
| 中途加入 | 仅在 LOBBY 阶段可加入。游戏开始后新玩家无法加入 |
| 跨世界 | 所有在线玩家均可 `/ig join`，不受世界限制 |

**管理员操作：**
- `/ig start` — 手动触发游戏开始（若人数不足会被拒绝）
- `/ig setspawn lobby` — 设置大厅生成点
- `/ig setspawn game 0` — 设置玩家自由行动出生点（可设多个，按索引）

---

### 阶段 1：倒计时 (STARTING)

**触发条件：** 管理员执行 `/ig start` 且大厅人数 ≥ `min-players`（默认 10）

**行为：**
1. 系统将大厅玩家注册到 `PlayerData` 中
2. 启动 60 秒倒计时（`lobby-wait-seconds` 可配置）
3. 每 10 秒广播倒计时提示；最后 5 秒每秒提示

**关键细节：**
| 细节 | 说明 |
|------|------|
| 玩家可见性 | 此阶段玩家仍在原地，**尚未分配角色**，无特殊能力 |
| 可取消 | 管理员 `/ig stop` 可中止，返回 LOBBY |
| 断线处理 | 若玩家在倒计时期间断线，将被移出大厅列表 |

---

### 阶段 2：身份揭示 (ROLE_REVEAL)

**触发条件：** 倒计时归零

**行为（按顺序）：**

1. **角色分配**
   - 系统将大厅玩家列表随机洗牌
   - 按 `config.yml` 配置分配：前 6 人为侦探团、中间 2 人为模仿者、最后 2 人为神秘客
   - 具体身份在阵营内部也随机洗牌（如 6 个侦探团角色中随机选）
   - 模仿者之间**不知道彼此身份**（当前版本无队友提示）

2. **个人信息揭示**
   - 每个玩家屏幕中央显示 3 秒 Title 字幕：角色名称 + 阵营
   - 聊天栏显示三条消息：
     - `你的身份是: [角色名]`（带阵营颜色）
     - `阵营: [阵营名]`
     - `能力: [能力描述]`

3. **能力物品发放**
   - 系统调用 `Role.giveAbilityItems(player)`，将能力物品放入玩家背包
   - 每个物品携带 `PersistentDataContainer` 标记（`ability` key → 角色名），用于系统识别
   - 特殊角色不需要物品：愚人、哨兵（被动能力）

**能力物品一览：**

| 角色 | 物品 | 外观（对其他玩家） |
|------|------|---------------------|
| 侦探 | 指南针 (Compass) | **可见**（指南针是普通物品，其他玩家可以看到你手中持有它） |
| 治安官 | 金剑 (Golden Sword) | **可见** |
| 猎人 | 钻石剑 (Diamond Sword) | **可见** |
| 锁匠 | 线 (String) | **可见**（但不显眼） |
| 香料师 | 烈焰粉 (Blaze Powder) | **可见** |
| 神偷 | 药水 (Potion) | **可见** |
| 千面人 | 玩家头颅 (Player Head) | **可见** |
| 阴谋家 | 书与笔 (Writable Book) | **可见** |
| 烟火师 | TNT | **可见**（非常明显！） |
| 流浪汉 | 纸 (Paper) | **可见**（但不显眼） |
| 送货员 | 命名牌 (Name Tag) | **可见**（但不显眼） |

**4. 系统初始化**
- `TaskManager`：为每位侦探分配 5 个任务
- `ScoreboardManager`：创建计分板并分配给所有玩家
- `TimedEventManager`：启动事件 tick 循环
- `MeetingManager` / `VotingManager`：初始化会议系统

**5. 倒计时过渡**
- 5 秒倒计时后自动进入自由行动阶段

---

### 阶段 3：自由行动 (FREE_ACTION)

这是游戏的核心阶段。所有存活玩家分散在地图中执行各自目标。

#### 3.1 进入自由行动

- 所有存活玩家被传送到预设的**游戏出生点**（`game-spawns` 配置）
- 玩家模式设为 **冒险模式 (ADVENTURE)**，禁止破坏方块
- 玩家背包被**清空后重新发放能力物品**
- 计分板开始每 2 秒刷新一次

#### 3.2 侦探团行动

**目标：** 完成所有任务 OR 投票淘汰全部模仿者

##### 任务系统
- 每位侦探分配到 5 个随机任务（`tasks-per-player` 配置）
- 任务分布在 config.yml 预设的交互点（按钮、拉杆、压力板、方块交互）
- 完成方式：右键点击目标方块（按钮/拉杆）或踩压力板
- 只有**该任务所属的侦探**才能完成它
- 完成时：播放经验球音效 + 聊天栏提示 `任务完成！(3/5)`
- **任务进度仅本人可见**，其他玩家无法知道你完成了多少

##### 任务完成=侦探团直接胜利
```java
if (totalTasks > 0 && completed >= totalTasks) {
    announceWin("侦探团");  // 游戏立即结束
}
```

##### 能力使用
- **侦探**：右键对任何玩家使用指南针 → 聊天栏显示该玩家的阵营颜色（蓝=侦探/黄=神秘客/红=模仿者）
  - 调查结果**仅自己可见**，不会公屏广播
- **治安官**：右键对任何玩家使用金剑 → 若目标是模仿者则目标死亡；若目标无辜则**治安官自己死亡**
  - 这是一次性击杀（无冷却，但用一次就可能导致自己死亡）
- **猎人**：右键使用钻石剑击杀 → 仅 1 次（`hunter-max-uses: 1`）
- **哨兵**：**被动能力**。被击杀时自动触发紧急会议。无主动物品。
- **锁匠**：右键铁门 → 开关秘密通道（仅模仿者和锁匠允许操作）
- **香料师**：右键使用烈焰粉标记玩家 → 被标记玩家获得发光效果 60 秒

#### 3.3 模仿者行动

**目标：** 使幸存者中模仿者人数 ≥ 50%（例如 4 人存活时 2 个模仿者即胜）

##### 击杀机制
- 模仿者没有内置的"右键击杀"通用能力。当前击杀通过特定角色能力实现：
  - **神偷**：右键触发 10 秒隐身 + 速度 II（接近目标后可用其他方式击杀）
  - **烟火师**：放置 TNT 炸弹，5 秒后爆炸，半径 3 格
  - **阴谋家**：在会议阶段猜身份杀人
- **当前限制：** 模仿者尚无统一的近战击杀能力。击杀主要依赖角色能力和会议。

##### 特殊事件触发
模仿者可以触发两大特殊事件来制造混乱：

**停电事件** (`triggerPowerOutage`)
- 效果：所有非模仿者获得失明 (Blindness) + 黑暗 (Darkness) 效果，持续 60 秒
- 模仿者视野**完全不受影响**
- 修复方式：任何玩家右键配电室修复点 → 提前解除
- 到期自动恢复
- 冷却：90 秒（同一事件不可叠加）

**水淹事件** (`triggerFlooding`)
- 效果：全局 BossBar 显示 75 秒倒计时
- 修复方式：两名玩家分别前往**东沐浴间**和**盥洗室**的两个阀门，右键交互
- **到期后果：模仿者直接获胜！**
- 若两阀门全部关闭：事件解除，模仿者计谋失败

**当前限制：** 事件触发入口尚未完全接入能力物品。需要通过 `/ig` 命令或特定方块交互触发。

#### 3.4 神秘客行动

**目标：** 完成独特个人目标（独立于阵营胜负）

- **愚人**：**被投票淘汰即胜利**。主动能力无。需要在会议中表现可疑让其他玩家投票你。
- **流浪汉**：右键与不同玩家互动（`vagabond-min-interactions: 5` 次）。每次互动计数。
- **送货员**：向所有玩家"投递包裹"（右键互动）。目标数为 `deliveryman-targets: 10`。

#### 3.5 自由行动阶段的限制与规则

| 规则 | 实现方式 | 细节 |
|------|----------|------|
| **禁止聊天** | `ChatListener` 拦截 `AsyncChatEvent` | 存活玩家发送消息 → 被取消 + 提示"只能在会议阶段发言" |
| **死亡玩家聊天** | 同上 | 死亡玩家消息仅转发给其他死亡玩家（`[死亡] 玩家名: 消息`），存活玩家看不到 |
| **禁止破坏方块** | 冒险模式 (ADVENTURE) | 所有存活玩家强制设为冒险模式 |
| **禁止移动（会议期间）** | `PlayerMoveListener` 取消移动事件 | 任何坐标变化都被拦截 |
| **禁止伤害（会议期间）** | `PlayerDeathListener.onDamage` 取消伤害 | 会议阶段玩家无敌 |
| **手持物品可见性** | Minecraft 原生机制 | **所有玩家手持物品对其他玩家完全可见**。这是一项核心设计约束——TNT 持有者极易被辨认 |
| **尸体标记** | ArmorStand 实体 | 玩家死亡后在原地生成隐形 ArmorStand，显示名称 `§c[玩家名] 的尸体` |

#### 3.6 计分板显示

侧边栏计分板每 2 秒刷新，显示内容：

```
§6§l模仿者游戏
§7==============
§a存活: 8/10
§e任务: 12/30
§7--------------
§b侦探: 5
§c模仿者: 2
§e神秘客: 1
§7==============
§a自由行动        ← 当前阶段
```

注意：阵营计数基于**存活玩家中**的数据，死亡玩家不被计入。这意味侦探可以通过观察计分板推测存活者阵营分布。

### 阶段 4：会议阶段

#### 4.1 触发会议的方式

**方式一：发现尸体**
1. 玩家右键点击尸体 ArmorStand
2. ArmorStand 被删除（尸体消失）
3. `callMeeting(reporter, victimUuid)` 被调用
4. 全局广播："[报告者] 发现了 [死者] 的尸体！"

**方式二：使用电台**
1. 玩家站在餐厅电台位置（5 格范围内）
2. 输入 `/radio` 命令
3. 每局每人**仅可使用一次电台**（`usedRadio` 标记）
4. 全局广播："[使用者] 通过电台召开了紧急会议！"

**方式三：哨兵被动**
1. 哨兵被击杀 → 自动触发 `callMeeting(null, victimUuid)`
2. 全局广播："哨兵被击杀！自动召开紧急会议！"

#### 4.2 会议进入

1. **保存位置**：所有存活玩家的当前位置被记录到 `preMeetingPosition`
2. **传送到会议室**：所有存活玩家传送到 `meeting-room` 预设位置
3. **冻结移动**：`PlayerMoveListener` 开始拦截移动事件
4. **设置速度**：`setWalkSpeed(0)` 禁止行走
5. **模式切换**：设为冒险模式（确保不破坏）

#### 4.3 讨论阶段 (MEETING_DISCUSSION)

- **时长：** 30 秒（`discussion-seconds` 配置）
- **聊天规则：**
  - 存活玩家可以自由发言
  - 消息经过渲染器处理，显示为 `[阵营颜色]玩家名: 消息`
  - 死亡玩家**不可发言**（消息被取消）
  - 死亡玩家可以看到存活玩家的讨论
- **提示节奏：** 20 秒、10 秒、最后 5 秒每秒提示剩余时间

**关键细节：**
- 所有存活玩家聚在同一会议室，可看到彼此
- **没有人知道其他人的真实身份**（除非侦探调查过）
- 玩家可以在这个阶段观察他人的手持物品——比如烟火师手持 TNT 将非常明显

#### 4.4 投票阶段 (MEETING_VOTING)

**时长：** 20 秒（`voting-seconds` 配置）

**投票方式：**
- `/vote <玩家名>` — 投票给指定存活玩家
- `/vote skip` — 弃权

**投票规则：**
- 每人仅一票
- 重复投票被拒绝
- 死亡的玩家不可投票
- **提前截止：** 所有存活玩家完成投票后，投票立即结束（不等满 20 秒）

**投票反馈：**
- 投票者：`"你投票给了 [目标]"` / `"你选择了弃权"`
- **其他玩家看不到你投给了谁**（投票是秘密的）

#### 4.5 结果统计（三条件淘汰判定）

投票结束后，`VotingManager.processResults()` 按以下逻辑判定：

```
条件 1: 最高票数 > 弃权人数
条件 2: 唯一最高票（无平票）
条件 3: 至少有人被投票（非全员弃权）

三个条件全部满足 → 淘汰最高票者
任一条件不满足 → 无人被淘汰
```

**具体场景：**

| 场景 | 票数分布 | 弃权 | 结果 |
|------|----------|------|------|
| 正常淘汰 | A:5 B:2 C:1 | 1 | A 被淘汰（5>1 且唯一最高） |
| 平票 | A:3 B:3 C:1 | 1 | **无人淘汰**（最高票 3 平票） |
| 票数不足 | A:2 B:1 C:1 | 4 | **无人淘汰**（最高票 2 ≤ 弃权 4） |
| 全员弃权 | - | 8 | **无人淘汰** |
| 无人投票 | - | 0 | **无人淘汰**（静默） |

**淘汰后：**
- 被淘汰者进入旁观者模式
- 全局广播："[玩家名] 被投票淘汰！"
- **不揭示被淘汰者的角色**（保持悬念）

#### 4.6 结果展示 (MEETING_RESULT)

- 10 秒结果展示时间
- 之后自动调用 `WinConditionChecker.checkAllConditions()`
- 若无获胜方 → `returnToFreeAction()` 回到自由行动

#### 4.7 会议结束 → 返回自由行动

1. 每个存活玩家传送回 `preMeetingPosition`（会议前位置）
2. 背包清空后重新发放能力物品
3. 投票状态重置（`resetForNewRound`）
4. 计分板恢复显示
5. 全局广播："会议结束，自由行动阶段继续！"

### 阶段 5：胜利判定与游戏结束 (GAME_OVER)

`WinConditionChecker` 在以下时机被调用：
- 每次会议结束后
- 每次任务完成后
- 水淹倒计时到期时

#### 5.1 侦探团胜利条件

**条件 A：任务全部完成**
```java
totalTasksCompleted >= targetTotalTasks  // 所有任务完成
→ "所有演绎任务已完成！侦探团获胜！"
```

**条件 B：所有模仿者被淘汰**
```java
所有模仿者都处于死亡状态
→ "所有模仿者已被淘汰！侦探团获胜！"
```

#### 5.2 模仿者胜利条件

**条件 A：人数优势**
```java
存活模仿者 * 2 >= 总存活人数  // 模仿者 ≥ 一半
→ "模仿者已占据多数！模仿者获胜！"
```

例如：
- 6 人存活，2 模仿者 → 2×2=4 ≥ 6? 否 → 未胜利
- 4 人存活，2 模仿者 → 2×2=4 ≥ 4? 是 → **胜利！**
- 3 人存活，2 模仿者 → 2×2=4 ≥ 3? 是 → **胜利！**

**条件 B：水淹到期**
```java
水淹倒计时结束且还有模仿者存活
→ "水淹事件到期！模仿者获胜！"
```

#### 5.3 神秘客个人胜利

| 角色 | 判定条件 |
|------|----------|
| 愚人 | 被投票淘汰（`isAlive() == false` 且被投过票） |
| 流浪汉 | 与不同玩家的互动次数 ≥ `vagabond-min-interactions`（默认 5） |
| 送货员 | 投递进度 ≥ `deliveryman-targets`（默认 10） |

神秘客胜利时：
- 胜利者本人：`"个人目标达成！你赢了！"`
- 其他玩家：`"神秘客个人胜利！游戏结束！"`（集体失败）

#### 5.4 游戏结束清理

```java
endGame():
  取消所有计时器
  停止计分板更新任务
  停止事件管理器（清除所有 Buff/BossBar/效果）
  移除计分板（玩家恢复默认计分板）
  清除所有玩家数据
  所有在线玩家恢复冒险模式 + 清空背包
  状态机 → LOBBY（可以开始下一局）
```

## 四、手持物品可见性 —— 核心设计考量

### 4.1 当前状态

**所有玩家的手持物品对其他玩家完全可见。** 这意味着：

1. **烟火师风险极高：** 手持 TNT 对所有人可见。一旦被看到持有 TNT，将在会议中被立即指认。
2. **猎人/治安官有威慑力：** 手持钻石剑/金剑意味着能杀人，会议中可能被怀疑。
3. **侦探指南针较隐蔽：** 指南针在 MC 中较为常见，但若有玩家跟随他人右键，行为上会被察觉。
4. **模仿者无统一击杀物品：** 当前模仿者（神偷、千面人）无即时击杀能力——神偷只能隐身，烟火师用 TNT。这在手持可见的环境下给模仿者造成很大劣势。

### 4.2 建议改进方向

| 问题 | 建议 |
|------|------|
| 物品暴露身份 | 添加"隐藏手持物品"机制——可在插件中设置 `player.setInvisible()` 或使用数据包阻止他人看到特定物品 |
| 模仿者击杀不足 | 为模仿者添加统一击杀能力（如金剑），仅在靠近玩家时通过代码触发击杀，而非依赖物品 |
| 部分职业物品无害 | 锁匠的线、流浪汉的纸不应引起怀疑——这些材料在冒险模式下"持有线"反而可疑（因为无合成功能） |

## 五、配置参数一览

所有参数在 `config.yml` 中可调：

```yaml
game:
  min-players: 10           # 最低开始人数
  role-distribution:
    detective-count: 6      # 侦探团人数
    imitator-count: 2       # 模仿者人数
    mystery-guest-count: 2  # 神秘客人数
  timers:
    lobby-wait-seconds: 60  # 大厅等待倒计时
    role-reveal-seconds: 5  # 身份揭示停留时间
    discussion-seconds: 30  # 会议讨论时间
    voting-seconds: 20      # 投票时间
    power-outage-seconds: 60 # 停电持续
    flooding-seconds: 75    # 水淹倒计时
  tasks-per-player: 5       # 每人任务数
  abilities:
    hunter-max-uses: 1      # 猎人可使用次数
    vagabond-min-interactions: 5  # 流浪汉互动目标
    deliveryman-targets: 10      # 送货员投递目标
    bomb-fuse-seconds: 5    # 炸弹引信时间
    bomb-radius: 3          # 炸弹爆炸半径
```
